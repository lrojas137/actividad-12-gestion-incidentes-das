# Comunicación del proceso a los usuarios

Los usuarios del sistema de registro de pacientes podrán reportar incidentes relacionados con errores, fallas, lentitud, problemas de acceso o solicitudes de mejora.

Para reportar un incidente, el usuario debe entregar la siguiente información:

- Descripción clara del problema.
- Módulo o función afectada.
- Pasos realizados antes de que ocurriera el incidente.
- Evidencia del error, como capturas de pantalla si aplica.
- Nivel de afectación en el uso del sistema.

El equipo de desarrollo revisará el incidente, lo clasificará según su tipo y prioridad, y realizará seguimiento hasta su solución y cierre.

Este proceso permite que los reportes sean claros, completos y fáciles de analizar.
